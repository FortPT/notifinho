# Notifinho v1.5.1 release notes

## Release summary

Notifinho v1.5.1 is a focused presentation and compatibility update for QNAP
Microsoft Teams notifications.

It preserves the existing QNAP parser, SMTP input, routing, Discord output,
Microsoft Teams output, and configuration format introduced in earlier
releases.

## QNAP Microsoft Teams card improvements

QNAP Adaptive Cards now include:

- a dedicated NAS identity icon;
- status-aware severity icons;
- category-specific icons;
- event-specific headings for test, storage, backup, security, power, update,
  disk, SMART, RAID, and other common event families;
- icon-prefixed operational-detail labels;
- human-readable event-type values;
- cleaner handling of Notification Center test-message metadata.

Raw values such as `test_message` are displayed as `Test Message`.

Redundant fields such as the QNAP Notification Center sentence describing that
a test message came from the NAS are omitted when the same information is
already represented by the card title and message.

## Compatibility

No configuration changes are required.

Existing deployments keep using:

- SMTP port `8025`;
- the `qnap` routing identity;
- existing Discord or Microsoft Teams output targets;
- the `webhook` configuration key.

Existing SMTP-only and UniFi-enabled deployments remain compatible.

## Upgrade

1. Pull the `1.5.1` or updated `latest` image.
2. Recreate the Notifinho container.
3. Verify that the application reports version `1.5.1`.
4. Send a QNAP Notification Center test message.
5. Confirm the richer Adaptive Card appears in the configured Teams channel.

## Rollback

Recreate the container using `fortpt/notifinho:1.5.0` or the equivalent GHCR
image. No configuration rollback is required.
