# Notifinho v2.3.4 release notes

Released 23 July 2026.

v2.3.4 is the final corrective WebUI release for source icon sizing,
page-refresh persistence, safe inactive-source removal, destination test
identity, automatic update awareness, and regional backup-time entry. It
retains the unified `config.yaml` model and platform schema 6.

## Highlights

- Refreshing the browser with F5 keeps the current WebUI page instead of
  returning to Overview. The URL hash remains authoritative and a
  session-scoped fallback restores the last valid page when the hash is absent.
- The top-right compact three-dot operations menu remains in place with
  **Check for updates** and administrator-only **Restart Notifinho** actions.
- Update checks run after authenticated startup, every six hours while the
  browser is active, when a stale tab becomes visible again, and on demand.
  Requests are credential-free and restricted by Content Security Policy to the
  official GitHub releases API.
- Source icons are now transparently presented with final tuned sizing:
  Notifinho slightly larger, Dell iDRAC much larger, UniFi Network much
  larger, UniFi Protect much larger, QNAP much larger, and Synology larger.
- Inactive source removal accepts the browser payload shape currently emitted by
  the Sources view and safely removes the source as long as no enabled exact
  route still uses it. Historical delivery records remain preserved.
- Destination-card tests continue selecting the best enabled non-wildcard route
  connected to that destination, so Supermicro, HPE iLO, Dell iDRAC, Home
  Assistant, Grafana, Portainer, Proxmox, UniFi, and Zabbix routes use their
  corresponding formatter instead of a generic Notifinho card.
- Scheduled backup time is entered and displayed using the selected global
  12-hour or 24-hour preference rather than the browser or operating system's
  native time-control locale. The API continues receiving canonical `HH:MM`.

## WebUI update-check boundary

The browser performs a read-only request to:

```text
https://api.github.com/repos/FortPT/notifinho/releases/latest
```

No Notifinho session cookie, API token, or referrer is sent. The result is
cached locally for six hours and is advisory only; Notifinho never pulls or
installs an image from the browser.

## HTTP and HTTPS access

v2.3.4 does not change the session-cookie or reverse-proxy contract introduced
in v2.3.2. Trusted-LAN HTTP and reverse-proxied HTTPS continue using the existing
`platform.secure_cookies`, `webui.public_url`, and `webui.enforce_https`
settings.

Page persistence uses only the same-origin URL hash plus browser
`sessionStorage`; it does not copy credentials between HTTP and HTTPS origins.
The GitHub update request explicitly omits credentials and is the only external
browser connection permitted by the WebUI Content Security Policy.

## Source removal safety

A source is removable only when there is no enabled exact route for that
source. An enabled wildcard route does not preserve the source's WebUI
inventory entry, because the wildcard remains valid without that stored
category entry. Existing delivery history is never deleted.

v2.3.4 broadens backend request parsing for removal so the action accepts the
current browser request shape using `source`, `id`, or `name` keys.

## Destination test selection

When several non-wildcard routes use the same destination, the WebUI prefers an
enabled route and then the highest semantic priority. If no specific route is
available, the test safely falls back to the generic Notifinho event.

## Direct managed-mount deployment

v2.3.4 does not change the managed NFS/SMB capability or mount contract from
v2.3.2. Deploy the existing opt-in override only when Notifinho must mount a
remote destination itself:

```bash
docker compose \
  -f compose.production.yaml \
  -f compose.managed-backups.yaml \
  config

docker compose \
  -f compose.production.yaml \
  -f compose.managed-backups.yaml \
  up -d
```

The normal production service remains non-root and capability-free. The managed
override retains root with `DAC_OVERRIDE`, `FOWNER`, and `SYS_ADMIN`, while
keeping `cap_drop: ALL` and `no-new-privileges:true`. Managed NFSv3 mounts retain
`nolock` so the read-only container does not require `rpc.statd` runtime files.

## Upgrade

1. Stop Notifinho and copy the complete `config`, `state`, `logs`, and secrets
   mounts.
2. Pull `fortpt/notifinho:2.3.4`.
3. Keep the existing production or managed-backup Compose combination.
4. Start Notifinho.
5. Force-refresh the browser once so the new static assets are loaded.
6. Complete the
   [v2.3.4 acceptance checklist](../v2.3.4-acceptance-checklist.md).

## Compatibility and rollback

v2.3.4 does not change platform schema 6 or the unified configuration model. A
direct image rollback to v2.3.3 is supported after stopping the container,
provided configuration and state remain one matched set. The optional
`webui.removed_sources` setting already exists in v2.3.2; no new configuration
migration is introduced by this release.
