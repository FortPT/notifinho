# v2.3.1 acceptance checklist

Complete this on VM-09 before upgrading production. Test with one administrator
and one normal user. Record any failed item before changing another setting.

## Release and session

- [ ] `/api/health` reports version `2.3.1`.
- [ ] SQLite reports schema 6; no schema migration is attempted from v2.3.0.
- [ ] With `secure_cookies: false` and `enforce_https: false`, HTTP login works.
- [ ] HTTP page refresh preserves the authenticated session.
- [ ] With secure cookies and HTTPS enforcement enabled, HTTP receives a 308
      redirect and HTTPS login works.

## Header and profile menu

- [ ] Connected is absent from the top right.
- [ ] Refresh is absent from the top right.
- [ ] Standalone Sign out is absent from the top right.
- [ ] The profile picture opens a dropdown containing Security and Sign out.
- [ ] Security opens Account security.
- [ ] Sign out revokes the session and returns to Login.

## Overview and notices

- [ ] Page subtitles are absent and the centered main page title is larger.
- [ ] Administrator Notice has no “Notify every user” subtitle.
- [ ] Sending a notice creates it immediately without a JavaScript error or F5.
- [ ] Updating a notice updates it immediately without F5.
- [ ] Ordinary announcements expose Edit/dismiss but no Resolve button.
- [ ] Update notices expose Resolve and open Updates.
- [ ] Error notices expose Resolve and open Audit Log.
- [ ] Cards read Active Sources, Active Destinations, and Active Routes.
- [ ] Counts include only enabled resources.

## Sources, inputs, and routing

- [ ] Sources appears in navigation.
- [ ] An administrator can change only a source tag.
- [ ] Moving Grafana from Services to Applications persists after refresh.
- [ ] Source category icons are recognisable and use distinct semantic colours.
- [ ] Home Assistant shows Home Assistant API.
- [ ] Supermicro, HPE iLO, and Dell iDRAC show their Redfish input names.
- [ ] Routes contains an Input column using the same names.
- [ ] Routing Flow shows only the input under the source name, not the tag name.
- [ ] An active flow animates in the delivery direction.
- [ ] A disabled flow is yellow and its arrows are stationary.
- [ ] An unhealthy flow is red and its crosses move vertically.
- [ ] Reduced-motion preference suppresses continuous animation.

## Destinations and delivery history

- [ ] Channel / destination aligns with Use Components v2.
- [ ] “Shown in Routing Flow…” is absent.
- [ ] Information deliveries are blue.
- [ ] Warning deliveries retain warning colour.
- [ ] Error/failure deliveries are red even after transport success.

## Audit and settings

- [ ] The Entries selector appears after the Audit table.
- [ ] Entries supports 25, 50, 100, 150, 250, and 500.
- [ ] Restart Notifinho has clear spacing after Regional Settings.

## Backups

- [ ] Creating an NFS target automatically enables managed mounts.
- [ ] `UNAS-01 | Backups`, `192.168.0.163`, and
      `/var/nfs/shared/Backups/Notifinho` save without expecting a host bind
      mount when the managed-backup override is active.
- [ ] Test mounts the NFS export and performs a real write/remove probe.
- [ ] A scheduled and a manual backup reach that destination.
- [ ] The 24-hour preference displays `02:00`.
- [ ] The 12-hour preference displays `2:00 AM`.
- [ ] The unprivileged Compose deployment gives a clear capability error for a
      managed remote mount.

## Account picture

- [ ] A previously accepted PNG, JPEG, or WebP opens the crop editor.
- [ ] The image can be dragged inside the circular crop.
- [ ] Zoom changes the framing without losing the centre point.
- [ ] Save updates the sidebar and account pictures immediately.
- [ ] The crop survives browser refresh and container restart.

## Final regression

- [ ] Existing users, destinations, routes, source tags, notices, history,
      audit entries, backup targets, and schedules survive restart.
- [ ] One real Home Assistant event records Home Assistant API.
- [ ] One real Redfish event records the vendor Redfish input.
- [ ] One real information event is blue in Delivery History.
- [ ] One disabled and one intentionally broken route show the required states.
- [ ] The complete automated test suite and GitHub Actions pass.
