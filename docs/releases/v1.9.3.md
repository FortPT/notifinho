# Notifinho v1.9.3 release notes

## Release summary

Notifinho v1.9.3 is a backwards-compatible Redfish presentation and
deduplication patch for multi-server environments. It uses the standard
subscription `Context` value as the hardware Host in Discord and Microsoft
Teams cards.

Existing Redfish endpoints, authentication, routes, and payload contracts
remain unchanged.

## Fixed

- Redfish subscription Context is preserved as normalized system metadata and
  displayed as Host in Discord and Microsoft Teams hardware cards.
- Duplicate fingerprints include the host and safe Redfish origin path, so
  identical event identifiers from different servers do not suppress each
  other.
- Empty `MessageArgs` arrays are no longer rendered as a recommended action.
  Explicit `Resolution` and `RecommendedAction` values remain supported.

## Upgrade from v1.9.2

1. Back up the production configuration and stack definition.
2. Wait for the v1.9.3 release workflow to publish and verify both registries.
3. Pull `fortpt/notifinho:1.9.3` or
   `ghcr.io/fortpt/notifinho:1.9.3`.
4. Change only the image tag and recreate the container, preserving mounts,
   ports, routes, and secrets.
5. Confirm the banner reports Notifinho 1.9.3 and both listeners are healthy.
6. Submit one native Redfish test per server and verify the correct Host and
   Teams target.

No configuration or secret rotation is required.

## Rollback

Pin the image back to `fortpt/notifinho:1.9.2` (or the equivalent GHCR image)
and recreate the container with unchanged configuration and secrets.

## Validation completed

- Passed 512 automated parser, API, formatter, routing, security, release, and
  backwards-compatibility tests.
- Added regression coverage for SRV-01 and SRV-02 Context values, independent
  duplicate keys, Discord and Teams Host fields, and empty recommended actions.
- Preserved parser behavior for Redfish payloads that omit Context.
