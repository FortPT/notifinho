# v2.3.0 acceptance checklist

Complete this on a test deployment before upgrading production. Use one
administrator and one newly created normal user. Record failures before making
another configuration change.

## Login and session

- [ ] The login page contains only the fields and essential setup guidance.
- [ ] “Welcome back” is absent; the larger logo and Notifinho name are centered.
- [ ] The HTTP URL redirects with 308 to the configured HTTPS URL.
- [ ] Login works after the redirect and invalid credentials show a clear error.
- [ ] Refreshing a page does not sign out a valid session.
- [ ] Sign out is visible at the right of the header and revokes the session.

## Overview and notices

- [ ] Sources, Destinations, Routes, Success, Requests, and Latest Activity load.
- [ ] The history range is beside Success and Requests and refreshes both.
- [ ] Administrator notice publishing appears immediately without F5.
- [ ] An administrator can edit and resolve a notice without F5.
- [ ] Ordinary notices can be dismissed per user.
- [ ] Error/update notices cannot be dismissed and clear after resolution.
- [ ] A new user initially sees only Notification operations and Mounted Configuration.
- [ ] That user receives later announcements only after their first login.
- [ ] Normal users can read the requested Overview panels but not admin controls.

## Sources and routing flow

- [ ] Sources use Servers, Services, Applications, Storage, or Controllers icons.
- [ ] Home Assistant/webhook events show HTTP when HTTP was used.
- [ ] Xen Orchestra/Mailrise email events show SMTP when SMTP was used.
- [ ] Active, disabled, and unhealthy routes are all present.
- [ ] Route arrows move in the delivery direction.
- [ ] Reduced-motion browser preference disables or reduces animation.
- [ ] Routes use semantic priority and expose no Order field.

## Destinations

- [ ] Each card shows its output type, destination name, and channel label.
- [ ] Card Send test delivers immediately with no confirmation dialog.
- [ ] Preview Send test uses the currently edited preview content.
- [ ] Channel and Discord v2-component fields are aligned.
- [ ] Output type cannot be changed after creation.
- [ ] Generate preview loads content without a redundant success toast.
- [ ] A failed test has a visible error and does not blank the workspace.

## Delivery History

- [ ] Titles show the device/host and event, not a raw parser key.
- [ ] Integration, severity/status, location/source, and input are readable.
- [ ] Description/body appears in its own box with correct spacing.
- [ ] Error and Failure are red even when Discord/Teams delivery succeeded.
- [ ] Successful non-error events are green.
- [ ] Home Assistant reports HTTP and Xen Orchestra email reports SMTP.
- [ ] Normal users can read shared Delivery History without mutation controls.

## Audit, account, and settings

- [ ] Run checks adds the health-check audit action immediately without F5.
- [ ] Audit defaults to 25 and supports 50, 100, 150, 250, and 500 rows.
- [ ] Both administrators and users can select the permitted page sizes.
- [ ] Choose picture opens the OS picker for PNG, JPEG, or WebP.
- [ ] The circular preview supports drag and zoom; the saved crop persists.
- [ ] Restart is visible only to administrators and requires confirmation/reason.
- [ ] Cancel performs no action; confirm creates an audit entry and restarts.
- [ ] Docker returns Notifinho to service; there is no Shutdown action.

## Inputs

- [ ] Inputs and Backups are separate navigation views.
- [ ] Each input shows its readable name, type, and enabled status.
- [ ] Enable/disable persists and reports whether restart is required.
- [ ] Changing one input does not alter unrelated inputs.

## Backup destinations

- [ ] The destination table shows Type, Name, Host, and path.
- [ ] A Local destination passes connection/write testing.
- [ ] An NFS destination validates its host/export and write access.
- [ ] An SMB destination validates its host/share and write access.
- [ ] SMB credentials are never returned or shown after saving.
- [ ] A failed network/write test gives a useful bounded error.
- [ ] Deleting a destination does not delete existing backup files.
- [ ] Default production works with a host-mounted NFS/SMB path and no privilege.
- [ ] If explicitly selected, managed-mount Compose works on the trusted test host.

## Backup execution

- [ ] A schedule can select a named destination and shows last/next execution.
- [ ] Run now creates exactly one integrity-manifested backup.
- [ ] The manual backup appears on the selected Local/NFS/SMB target.
- [ ] The scheduled run occurs and appears on the same target.
- [ ] The copied backup passes its manifest/integrity verification.
- [ ] Duplicate clicks cannot overlap the same run.
- [ ] A failure creates an administrator error notice and Audit Log result.
- [ ] A later successful run/check clears the lifecycle error notice.
- [ ] Retention removes only backups owned by the applicable job.

## Final source validation

- [ ] One real Home Assistant event creates one HTTP route/send/history sequence.
- [ ] One real Xen Orchestra email creates one SMTP route/send/history sequence.
- [ ] Existing destinations, routes, applications, users, and history survive restart.
- [ ] One failed component leaves other workspace sections usable with a warning.
- [ ] Administrator mutations remain unavailable to normal users at the API.
