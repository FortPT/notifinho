# Notifinho v2.3.5 release notes

Released 23 July 2026.

v2.3.5 corrects the remaining source identity, Overview icon sizing, and
inactive-source removal issues reported after v2.3.4. Platform schema 6 and the
unified `config.yaml` model remain unchanged.

## Fixed

- The `xo` source key now uses the packaged Xen Orchestra icon and Xen
  Orchestra display name instead of the generic Notifinho fallback.
- `redfish`, `restful`, and `rest_api` source keys use a packaged neutral REST
  API icon instead of the generic fallback.
- Overview icons now use a source-key data attribute rather than fragile image
  URL substring matching.
- Notifinho is larger in Overview. Dell iDRAC, UniFi Network, UniFi Protect,
  and QNAP are substantially larger, while Synology is also enlarged.
- Inactive source removal now uses
  `DELETE /api/v2/source-categories/<source>` and therefore no longer depends on
  a JSON body surviving a browser, reverse proxy, or HTTP server DELETE request.
- The previous JSON-body DELETE endpoint remains available for compatibility.

## HTTP and HTTPS access

v2.3.5 does not change the trusted-LAN HTTP or reverse-proxied HTTPS session
contract. The new source-removal URL remains same-origin, CSRF-protected, and
administrator-only.

## Direct managed-mount deployment

v2.3.5 does not change the managed NFS/SMB backup override. Continue using the
existing production and managed-backup Compose files documented for v2.3.4.

## Upgrade

1. Back up the complete configuration, state, logs, and secrets mounts.
2. Pull `fortpt/notifinho:2.3.5`.
3. Redeploy with the same Compose configuration.
4. Force-refresh the WebUI once.
5. Complete the
   [v2.3.5 acceptance checklist](../v2.3.5-acceptance-checklist.md).

## Compatibility and rollback

v2.3.5 does not change the database schema or configuration schema. A direct
rollback to v2.3.4 remains supported when configuration and state are retained
as one matched set.
