# Notifinho v1.9.5 release notes

## Release summary

Notifinho v1.9.5 completes the Teams and Discord presentation work started in
v1.9.4. Every integration now uses an exact official vendor asset rather than
a generated initials badge. Teams and Discord share the same device/event,
context, message, metric, detail, status, and source-time semantics while
Discord retains its richer source-specific fields.

Existing valid webhooks, routes, endpoints, secrets, parsers, and source-time
behavior remain compatible. No secret rotation is required.

## Changed

- Added official, locally hosted assets for Xen Orchestra, Grafana, Portainer,
  Proxmox VE, QNAP, Synology, TrueNAS, UniFi Network, UniFi Protect, UniFi
  Drive, Zabbix, Redfish, Supermicro, HPE, Dell, Home Assistant, and generic
  Notifinho events.
- Normalized each asset to a transparent 256 px PNG suitable for the 48 px
  Teams header slot.
- Recorded the official source and transformation notes for every asset in
  `assets/icons/README.md`.
- Added the optional `NOTIFINHO_ICON_BASE_URL` environment override for
  immutable branch previews or installations that mirror the same assets.
- Preserved infrastructure identifiers and acronyms such as `PVE-01`, `CPU`,
  and `VMID` instead of title-casing them.
- Removed duplicated UniFi state, source context, and pin icons, and shortened
  the Network last-device label.
- Standardized all 17 Discord presentations on the same hierarchy used by
  Teams: `device • event`, integration/state/source context, event message,
  Severity/Category/Event time metrics, rich details, and official thumbnail.
- Added a shared Discord renderer that preserves integration-specific fields
  while enforcing the 25-field and 6,000-character embed limits.

## Fixed

- Xen Orchestra cards use the triggered backup name, omit missing Duration or
  Result values, and show explicit results such as `3 of 3 VMs successful` in
  both outputs. Per-VM size, speed, repository, and errors remain available in
  Discord.
- Placeholder values such as `PASTE_HERE`, malformed URLs, credential-bearing
  URLs, and non-HTTPS Teams webhooks are rejected before an outbound request.

## Upgrade from v1.9.4

1. Back up the production configuration and stack definition.
2. Pull `fortpt/notifinho:1.9.5` or `ghcr.io/fortpt/notifinho:1.9.5` only after
   the release workflow succeeds.
3. Change only the image tag and recreate the container, preserving mounts,
   ports, routes, secrets, and proxy configuration.
4. Confirm the banner reports Notifinho 1.9.5 and both listeners are healthy.
5. Send one representative event for every configured Teams and Discord
   integration and verify its official logo, hierarchy, source time, and
   optional details.

## Rollback

Pin the image back to `fortpt/notifinho:1.9.4` (or the equivalent GHCR image)
and recreate the container with unchanged configuration and secrets.

## Validation required before release

- The complete automated parser, API, formatter, routing, security, release,
  and backwards-compatibility suite passes: 626 tests.
- A branch preview must be deployed on the office VM and all 17 Teams
  presentations must be delivered and visually reviewed before merge or tag.
- The same branch preview must deliver all 17 Discord presentations to the
  VM-04 development destination for visual review before merge or tag.
