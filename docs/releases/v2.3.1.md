# Notifinho v2.3.1 release notes

Released 23 July 2026.

v2.3.1 is a corrective release for the WebUI, input presentation, managed
backup mounting, and profile-picture workflow reported after v2.3.0. It keeps
the unified `config.yaml` model and platform schema 6.

## Highlights

- Direct HTTP login works on a trusted LAN when
  `platform.secure_cookies: false`; HTTPS redirection now requires the explicit
  `webui.enforce_https: true` setting.
- Connected, Refresh, and the standalone header Sign out control are removed.
  The sidebar profile picture opens a menu containing Security and Sign out.
- Administrator notice create/update no longer calls `.reset()` through an
  expired browser event reference, so the notice appears immediately without
  F5.
- System update notices link to Updates; system errors link to Audit Log.
  Ordinary announcements retain Edit and per-user dismissal.
- Sources has its own view. Administrators can change only the source tag
  (Servers, Services, Applications, Storage, or Controllers).
- Routes and Routing Flow show the same readable input names, including Home
  Assistant API and vendor-specific Redfish inputs.
- Active Sources, Active Destinations, and Active Routes use enabled server-side
  counts.
- Disabled flows are yellow and stationary. Unhealthy flows are red and show
  vertically moving crosses. Information deliveries are blue.
- Audit row selection is at the bottom; Restart Notifinho has independent
  spacing from Regional Settings.
- Saving an NFS or SMB destination in the WebUI enables managed mounts
  automatically. The container must still be started with
  `compose.managed-backups.yaml`.
- Scheduled-backup presentation follows the global 12/24-hour preference.
- Profile pictures use bitmap decoding with a data-URL fallback before the
  drag/zoom circular crop opens.

## HTTP and HTTPS login

Trusted private-network HTTP:

```yaml
platform:
  secure_cookies: false

webui:
  public_url: ""
  enforce_https: false
```

Internet-facing or reverse-proxied HTTPS:

```yaml
platform:
  secure_cookies: true

webui:
  public_url: "https://notifinho.example.com"
  enforce_https: true
```

Do not expose the HTTP mode to an untrusted network. Passwords and session
cookies are not protected by transport encryption in that mode.

## Managed NFS/SMB destinations

The WebUI now changes `platform.backups.managed_mounts` to `true` before saving
a remote target. Linux still requires the mount helpers, root identity, and
`SYS_ADMIN` capability supplied by the opt-in override:

```bash
docker compose \
  -f compose.production.yaml \
  -f compose.managed-backups.yaml \
  config

docker compose \
  -f compose.production.yaml \
  -f compose.managed-backups.yaml \
  up -d
```

Without that override, Notifinho reports a bounded permission/mount failure
instead of pretending the remote target was tested.

## Upgrade

1. Complete the remaining v2.3.0 release verification first.
2. Stop Notifinho and copy the complete configuration and state mounts.
3. Pull `fortpt/notifinho:2.3.1`.
4. Apply the HTTP or HTTPS settings above.
5. If using remote targets, deploy both Compose files.
6. Start Notifinho and complete the
   [v2.3.1 acceptance checklist](../v2.3.1-acceptance-checklist.md).

## Compatibility and rollback

v2.3.1 does not change schema 6. A direct image rollback to v2.3.0 is supported
after stopping the container, provided no operator separately replaces the
configuration or state with older copies. Always retain an offline copy before
the upgrade.
