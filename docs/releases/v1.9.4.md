# Notifinho v1.9.4 release notes

## Release summary

Notifinho v1.9.4 standardizes Microsoft Teams presentation across every
integration and makes event-time behavior deterministic for worldwide
deployments. Teams and Discord now preserve the date and wall-clock time
emitted by the source machine and display recognized timestamps as
`20 Jul 2026 • 18:09`.

Existing inputs, authentication, routes, output targets, webhooks, and product
image URLs remain unchanged.

## Changed

- All Teams integrations now use one shared hierarchy:
  `device • event`, integration/state/source context, emphasized message,
  horizontal Severity/Category/Event time metrics, icon-labelled details,
  optional actions, and the Notifinho footer.
- Critical, warning, resolved, and informational states use consistent icons
  and Teams colors. A resolved state takes precedence over its previous
  severity, so recovery cards remain green.
- Teams and Discord preserve source wall-clock timestamps without converting
  them to the container, recipient, or UTC timezone.
- Visible UTC and numeric-offset suffixes were removed.
- When no source timestamp exists, Teams displays `—` and Discord omits the
  optional time field. Notifinho never substitutes its receipt time.
- The presentation contract is documented for future integrations and the
  planned WebUI timezone controls.

## Upgrade from v1.9.3

1. Back up the production configuration and stack definition.
2. Wait for the v1.9.4 release workflow to publish and verify both registries.
3. Pull `fortpt/notifinho:1.9.4` or
   `ghcr.io/fortpt/notifinho:1.9.4`.
4. Change only the image tag and recreate the container, preserving mounts,
   ports, routes, secrets, and proxy configuration.
5. Confirm the banner reports Notifinho 1.9.4 and both listeners are healthy.
6. Send representative events for the configured integrations and review the
   Teams layout, device identity, status color, details, and source time.

No configuration or secret rotation is required.

## Rollback

Pin the image back to `fortpt/notifinho:1.9.3` (or the equivalent GHCR image)
and recreate the container with unchanged configuration and secrets.

## Validation completed

- Passed 530 automated parser, API, formatter, routing, security, release, and
  backwards-compatibility tests.
- Added cross-integration assertions for the shared Teams hierarchy.
- Covered source-wall-clock preservation for naive ISO, offset-bearing ISO,
  seconds, and millisecond epoch timestamps.
- Verified that recovery state wins over historical critical severity.
