# v2.5.0 acceptance checklist

## Automated validation

- [ ] Complete Python test suite passes.
- [ ] JavaScript syntax validation passes.
- [ ] `git diff --check` passes.
- [ ] Production Docker image builds and reports v2.5.0.
- [ ] Platform state initializes at schema 8.

## Migration safety

- [ ] A matched pre-upgrade backup of config, state, and secrets exists.
- [ ] Destination, route, API-token, and settings counts match the migration preview.
- [ ] Existing application-token values still authenticate.
- [ ] Existing Discord and Microsoft Teams credentials remain configured.
- [ ] The migrated `config.yaml` retains its owner and file mode.
- [ ] The normalized file contains `platform_database_v1` and none of the moved sections.

## WebUI-managed settings

- [ ] XO outcome and ID preferences can be edited.
- [ ] Zabbix problem-ID visibility can be edited.
- [ ] Dell iDRAC trusted audit-client IPs can be edited.
- [ ] UniFi Protect device aliases can be edited.
- [ ] Home Assistant endpoint and component aliases can be edited.
- [ ] Redfish deduplication window can be edited.
- [ ] Regional settings and backup scheduling persist after restart.
- [ ] API applications can be created, rotated, enabled, disabled, and deleted.

## Fault isolation

- [ ] Destinations and routes load without parsing `config.yaml` resources.
- [ ] One invalid destination row does not hide valid destinations or routes.
- [ ] One invalid route row does not hide valid routes or destinations.
- [ ] One invalid settings row shows a scoped repair message and safe default.
- [ ] Unexpected API errors include an endpoint, code, and reference ID.

## Delivery regression

- [ ] One Discord test notification arrives at the expected channel.
- [ ] One Microsoft Teams test notification arrives at the expected workflow/channel.
- [ ] Zabbix SMTP and HTTP routes match only their configured input.
- [ ] Dell iDRAC, HPE iLO, Supermicro, and generic Redfish delivery remains functional.

## Rollback

- [ ] The operator has the exact pre-upgrade image tag and matched backup path.
- [ ] Rollback instructions explicitly restore config, state, and secrets together.
