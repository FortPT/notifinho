# Contributing

First of all, thank you for considering contributing to Notifinho!

Whether you're fixing a typo, improving the documentation, reporting a bug or implementing a new parser, every contribution is appreciated.

---

## Getting Started

1. Fork the repository.
2. Create a feature branch from `master`.
3. Make your changes.
4. Test your changes.
5. Submit a Pull Request.

---

## Coding Style

- Follow PEP 8.
- Keep functions focused and readable.
- Write meaningful commit messages.
- Prefer small, focused pull requests.

---

## Reporting Issues

When opening an issue, please include:

- Notifinho version
- Python version
- Docker version (if applicable)
- Steps to reproduce
- Expected behavior
- Actual behavior
- Relevant logs

---

## Feature Requests

Feature requests are welcome.

Before opening a new request, please check whether a similar proposal already exists.

---

## Questions

If you're unsure about a design decision, feel free to open a discussion before starting implementation.

---

Thank you for helping improve Notifinho.
