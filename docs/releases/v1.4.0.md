# Notifinho v1.4.0 release notes

## Highlights

### TrueNAS 26

Notifinho v1.4.0 adds provisional first-class support for TrueNAS 26
alert-service email.

TrueNAS notifications now receive dedicated source detection, parsing,
classification, Discord embeds, Microsoft Teams Adaptive Cards, and routing.

Supported alert layouts include:

- test alerts;
- new alerts;
- current alerts;
- cleared alerts;
- grouped notifications;
- plain-text, HTML, and multipart messages; and
- malformed-message fallback handling.

See the [TrueNAS integration guide](../truenas.md) for configuration, replay
commands, validation details, and current limitations.

### TrueNAS event classification

Recognizable TrueNAS events are normalized into the existing Notifinho
notification model and classified into these areas:

- storage pools and ZFS health;
- disks and SMART warnings;
- scrub failures;
- replication, backup, snapshot, and task failures;
- UPS, battery, and power conditions;
- system and hardware conditions;
- network conditions;
- security and certificate conditions;
- applications and services; and
- generic TrueNAS alerts when no stronger category is available.

### Source-aware detection

TrueNAS detection uses corroborating sender, header, subject, body, and
alert-structure signals.

The generic subject `Alerts` is not sufficient by itself. This protects
existing QNAP, Grafana, Xen Orchestra, Zabbix, and generic SMTP processing from
false TrueNAS classification.

### Discord and Microsoft Teams

TrueNAS notifications use dedicated Discord and Microsoft Teams formatters.

The formatters provide:

- system hostname;
- alert category;
- status and severity;
- alert count;
- event time when available;
- grouped alert details; and
- bounded payload sizes for platform limits.

### Dedicated routing

TrueNAS can use a dedicated webhook target or the existing default target:

```yaml
outputs:
  discord:
    default:
      webhook: "PASTE_DISCORD_WEBHOOK_HERE"

    truenas:
      webhook: "PASTE_TRUENAS_DISCORD_WEBHOOK_HERE"

routing:
  truenas:
    outputs:
      - output: discord
        target: truenas

```

Equivalent Microsoft Teams targets can be configured under `outputs.teams`.

The public
[`config/config.example.yaml`](../../config/config.example.yaml)
contains the complete examples. Existing private `config/config.yaml` files
must be updated locally and must not be committed.

## Test email and test alert behavior

TrueNAS provides two related tests:

- **Send Test Email** validates the global SMTP transport and can safely use
  Notifinho's generic fallback.
- **Send Test Alert** validates the TrueNAS Email alert-service format and is
  detected by the TrueNAS integration.

Both private TrueNAS 26 samples were replayed successfully during development.
A fresh live Send Test Alert was also detected, parsed, routed, formatted, and
delivered successfully.

## Test coverage

The v1.4.0 suite contains 169 passing tests.

Coverage includes:

- TrueNAS source detection and precedence;
- plain-text, HTML, and multipart parsing;
- test, new, current, cleared, grouped, and malformed messages;
- category, status, severity, hostname, and event-time normalization;
- Discord and Microsoft Teams payload generation;
- aggregate payload limits;
- output formatter selection;
- routing regressions; and
- protection for existing notification sources.

Nine anonymized synthetic TrueNAS fixtures are included for offline regression
testing.

## Upgrade notes

- The application version is now `1.4.0`.
- Compare the updated public configuration examples with the deployment's
  private `config/config.yaml`.
- Add `routing.truenas` only when TrueNAS delivery is required.
- A dedicated `outputs.discord.truenas` or `outputs.teams.truenas` target is
  optional; routes can continue to use `default`.
- Existing Xen Orchestra, Zabbix, QNAP, Grafana, generic SMTP, and conditional
  routes remain supported.
- Development SMTP uses host port `8026`; production SMTP continues to use
  port `8025`.

## Provisional compatibility

TrueNAS support remains provisional for broader real-world alert variants,
localized wording, customized templates, Enterprise HA layouts, and future
TrueNAS message-format changes.

Private `.eml` samples can contain hostnames, addresses, pool and dataset
names, disk serial numbers, usernames, routing headers, and recipient
addresses. Original private messages must remain outside Git.

versioned and `latest` images to Docker Hub and GitHub Container Registry, and
creates or updates the corresponding GitHub Release.
