# v2.4.0 acceptance checklist

## Sources

- [ ] Every built-in integration appears on a clean installation.
- [ ] Dell iDRAC expands to `Redfish` without displaying `dell_idrac`.
- [ ] Zabbix expands to `SMTP` and `HTTP`.
- [ ] There is no source status or removal control.
- [ ] Category changes are saved at integration level.
- [ ] Home Lab Generic is not presented as an integration.

## Routes

- [ ] Add/edit contains Zabbix (SMTP), Zabbix (HTTP), Generic (HTTP), and Generic (Redfish).
- [ ] Existing routes retain destinations, filters, priorities, and enabled state.
- [ ] SMTP events do not match HTTP-only routes and vice versa.

## Destinations

- [ ] A duplicate name returns HTTP 409 and leaves YAML unchanged.
- [ ] The Add button is disabled while a save is pending.
- [ ] Enabling a destination also enables its parent output provider.
- [ ] Destination type can be changed with new credentials.
- [ ] Routes and history retain the same destination ID after a type change.
- [ ] Send test reports the provider response or explicit error.

## Safety

- [ ] `config.yaml` owner, group, and mode are preserved.
- [ ] A failed sync restores the prior YAML.
- [ ] Internal errors show a reference also present in container logs.
- [ ] VM-09 isolated test mounts no VM-04 production directory.
