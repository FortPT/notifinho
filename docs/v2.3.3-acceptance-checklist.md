# v2.3.3 acceptance checklist

Complete this checklist on VM-09 before merge and on VM-04 before considering
the production rollout complete.

## Automated release gate

- [ ] The complete Python suite passes.
- [ ] Python compilation, JavaScript syntax, documentation metadata, container
      asset validation, and `git diff --check` pass.
- [ ] `/api/health` reports version `2.3.3`.
- [ ] The release candidate starts with platform schema 6.
- [ ] `src/webui/enhancements.js` and `enhancements.css` are served from the
      built image.

## Page reload and navigation

- [ ] Open Destinations, press F5, and remain on Destinations.
- [ ] Repeat for Sources, Updates, Backups, Settings, and Audit log.
- [ ] Copy a `#backups` URL into a new authenticated tab and open Backups.
- [ ] An invalid page hash safely returns to Overview.
- [ ] Administrator-only pages cannot be restored for a normal user.

## Header operations menu

- [ ] The standalone Restart button is absent.
- [ ] The top-right three-dot button opens a menu matching the intended compact
      Home Assistant interaction.
- [ ] Check for updates is visible to authenticated users.
- [ ] Restart Notifinho is visible only to administrators.
- [ ] Restart still requires a reason, records an audit event, and returns after
      the container restart policy restarts the service.

## Source icons and safe removal

- [ ] Overview and Sources show transparent, larger vendor icons without white
      tiles.
- [ ] An inactive source shows Remove only to an administrator.
- [ ] Cancel leaves the source unchanged.
- [ ] An enabled wildcard route does not prevent removing an otherwise inactive
      exact source.
- [ ] An enabled exact route returns a clear conflict and leaves the source.
- [ ] Confirmed removal hides the source, clears its saved category, records
      `source.remove`, and retains historical delivery rows.

## Destination tests

- [ ] Send test on Charlie's destination reaches `#supermicro-alerts`.
- [ ] The resulting Discord notification uses the Supermicro formatter/card,
      provider identity, and hardware details.
- [ ] HPE iLO, Dell iDRAC, Home Assistant, Grafana, Portainer, Proxmox, UniFi,
      and Zabbix destinations use their corresponding formatter when routed.
- [ ] A destination without a specific route uses the generic Notifinho test.
- [ ] The selected destination and channel are never changed by the test.

## Update checks

- [ ] An authenticated startup performs or restores a recent update check.
- [ ] Manual Check for updates reports current or available version clearly.
- [ ] The Updates page records the last check time.
- [ ] A failed GitHub request leaves the WebUI usable and shows a bounded error.
- [ ] Browser developer tools show no Notifinho cookie or token sent to GitHub.
- [ ] A stale visible tab checks again after six hours.

## Backup clock

- [ ] With 24-hour selected, the field displays `14:20` and never `02:20 PM`.
- [ ] With 12-hour selected, the field displays `02:20 PM`.
- [ ] Saving either representation stores canonical `14:20` in configuration.
- [ ] Invalid time text is rejected before the request is sent.
- [ ] Last scheduled run timestamps still follow timezone and clock preference.

## Publication and production

- [ ] GitHub Actions passes on the v2.3.3 draft PR.
- [ ] The PR is merged before tag `v2.3.3` is created.
- [ ] The GitHub Release uses `docs/releases/v2.3.3.md`.
- [ ] Docker Hub and GHCR expose both `2.3.3` and matching `latest` images.
- [ ] VM-04 is pinned to `fortpt/notifinho:2.3.3`.
- [ ] HTTP, HTTPS, SMTP, Redfish/Home Assistant inputs, destination routing, and
      scheduled NFS backup remain healthy after the production restart.
