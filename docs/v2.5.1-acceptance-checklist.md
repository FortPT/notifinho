# v2.5.1 acceptance checklist

## Release candidate

- [ ] Complete Python test suite passes on VM-09.
- [ ] JavaScript syntax validation passes.
- [ ] Docker image reports v2.5.1 and schema 8.
- [ ] Copied production counts for destinations, routes, tokens, users, and
      settings match the baseline.

## Fallback routing

- [ ] A real Dell iDRAC event reaches `idrac-alerts` exactly once.
- [ ] The same iDRAC event does not reach `default-alerts`.
- [ ] An unmatched synthetic HTTP event reaches the HTTP fallback route.
- [ ] An unmatched synthetic Redfish event reaches the Redfish fallback route.
- [ ] Two matching routes to the same destination create one final delivery.
- [ ] Include and exclude filters behave as labelled; exclude wins.

## Sources and settings

- [ ] Sources show only SMTP, HTTP, and Redfish input badges.
- [ ] Zabbix appears once with SMTP and HTTP.
- [ ] No “events received through” description is displayed.
- [ ] Integration behavior cards have visible spacing between name, summary,
      badge, and Edit action.
- [ ] Xen Orchestra settings show only the job/run ID formatter option.
- [ ] Existing XO routes contain the migrated skipped/failure status filters.
- [ ] Inputs lists exactly SMTP, HTTP, and Redfish.
- [ ] Disabling Redfish prevents delivery while safely acknowledging Redfish subscriptions.
- [ ] An enabled input has a green source line and green first flow arrow.
- [ ] A disabled input has a yellow source line and yellow prohibition symbol.
- [ ] A missing or invalid input has a red source line and red X symbol.
- [ ] Destination status remains independently green when the input or route is disabled.
- [ ] The Regional Settings and Integration Behavior panels have a visible gap.

## Destinations and Event API tokens

- [ ] Enabled/Disabled is clickable and uses green/red state styling.
- [ ] Shared/Private is clickable and uses green/neutral state styling.
- [ ] Credentials set is absent; missing credentials are clearly flagged.
- [ ] A successful destination test displays Last test passed.
- [ ] API access is labelled Event API tokens and explains `/api/v2/events`.
- [ ] Issue token lists all built-in integrations and accepts custom source IDs.
- [ ] Existing tokens still authenticate only against `/api/v2/events`.

## Production

- [ ] Durable pre-upgrade baseline is available or validator reports comparison
      unavailable without skipping health checks.
- [ ] Discord destination test passes.
- [ ] Microsoft Teams destination test passes.
- [ ] No severe migration notice appears.
- [ ] Production container restart count remains zero.
