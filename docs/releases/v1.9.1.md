# Notifinho v1.9.1 release notes

## Release summary

Notifinho v1.9.1 is a backwards-compatible presentation patch for the v1.9
event platform. It gives generic authenticated API events their own Discord
and Microsoft Teams presentation and makes Home Assistant cards concise,
structured, and service-aware.

This release does not change the configuration schema, authentication model,
routes, or endpoint contracts introduced in v1.9.0.

## Fixed

- Unknown and authenticated API event sources now use dedicated generic
  Discord and Microsoft Teams formatters instead of falling back to Xen
  Orchestra backup presentation.
- Source `xo` remains explicitly mapped to the existing Xen Orchestra
  formatters.
- Home Assistant system-log events now derive the responsible service,
  device, entity, endpoint, and retry information where available.
- Home Assistant Event text is concise and bounded. Python source paths and
  verbose internal service-object representations are omitted from cards.
- Explicit fields supplied by purpose-built Home Assistant automations remain
  authoritative and are not replaced by derived values.

## Home Assistant configuration

The documented Home Assistant example is intentionally transport-focused. A
generic automation can forward selected `system_log_event` payloads while
Notifinho handles reusable normalization and presentation. Environment-only
filters, ignored devices, and noise suppression remain local Home Assistant
policy.

Existing `notifinho.home_assistant.v1` automations remain compatible. Adopting
the simplified example is optional.

## Upgrade from v1.9.0

1. Back up the production configuration, mounted token files, and current
   stack definition.
2. Wait for the `v1.9.1` release workflow to pass and publish both registries.
3. Pull `fortpt/notifinho:1.9.1` or
   `ghcr.io/fortpt/notifinho:1.9.1`.
4. Change only the image tag and recreate the container, preserving mounts,
   ports, routes, and secrets.
5. Confirm the banner reports Notifinho 1.9.1, both configured listeners are
   healthy, and unauthenticated HTTP requests are still rejected.
6. Send one authenticated generic event and one Home Assistant event, then
   confirm the expected destination and readable card fields.

No configuration migration or token rotation is required.

## Rollback

Pin the image back to `fortpt/notifinho:1.9.0` (or the equivalent GHCR image)
and recreate the container with the unchanged configuration and secrets.

Events accepted while v1.9.1 is running use the same contracts as v1.9.0, so
rollback does not require payload or automation changes.
